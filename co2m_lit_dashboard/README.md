# CO2M Literature Intelligence Dashboard

A build-once, run-independent Python pipeline that extracts and analyses research PDFs, then serves a Streamlit dashboard — no LLMs, no external APIs.

---

## Project structure

```
co2m_lit_dashboard/
├── config/
│   └── concepts.yaml          # Concept taxonomy (editable)
├── src/
│   ├── build_corpus.py        # Pipeline entry point (CLI)
│   ├── pdf_extractor.py       # PyMuPDF text & metadata extraction
│   ├── metadata_loader.py     # Load CSV/JSON external metadata
│   ├── concept_classifier.py  # Keyword-based chunk classification
│   ├── summarizer.py          # Extractive concept summarisation
│   ├── visual_tables.py       # Pre-compute visualisation tables
│   └── storage.py             # Parquet save/load + manifest
└── dashboard/
    └── app.py                 # Streamlit dashboard

co2m/
├── pdfs/                      # Input PDFs  ← put your files here
├── metadata/                  # Optional CSV/JSON metadata
└── artifacts/                 # Generated by the build step (Parquet + JSON)
```

## Quick start

### 1 – Install dependencies

```bash
pip install -r co2m_lit_dashboard/requirements.txt
```

### 2 – Place PDFs

Copy research PDF files into `co2m/pdfs/`.

### 3 – (Optional) add metadata

Drop any CSV or JSON metadata files into `co2m/metadata/`.  
Supported columns (case-insensitive): `filename`, `title`, `authors`, `year`.

### 4 – Run the build pipeline

```bash
python co2m_lit_dashboard/src/build_corpus.py \
    --pdf-dir      co2m/pdfs \
    --metadata-dir co2m/metadata \
    --out          co2m/artifacts
```

The build writes these files into `co2m/artifacts/`:

| File | Description |
|------|-------------|
| `documents.parquet` | One row per PDF (title, authors, year, page count, hash) |
| `pages.parquet` | Raw text per page |
| `chunks.parquet` | Sliding-window text chunks |
| `concepts.parquet` | Chunk → concept assignments with confidence scores |
| `summaries.parquet` | Extractive summary per concept |
| `concept_counts.parquet` | Chunk & doc counts per concept |
| `top_documents_by_concept.parquet` | Ranked docs per concept |
| `keywords_by_concept.parquet` | TF-IDF keyword weights per concept |
| `manifest.json` | Build metadata |

### 5 – Launch the dashboard

```bash
streamlit run co2m_lit_dashboard/dashboard/app.py
```

The dashboard reads **only** from `co2m/artifacts/`. You can safely delete
`co2m/pdfs/` afterwards and the dashboard will still work.

## Customising the taxonomy

Edit `co2m_lit_dashboard/config/concepts.yaml` to add, rename, or extend
concepts.  Add more keywords under each concept to improve classification
recall, then re-run the build step.

```yaml
My New Concept:
  - keyword one
  - keyword two
  - keyword phrase
```

## CLI options

```
python co2m_lit_dashboard/src/build_corpus.py --help

options:
  --pdf-dir       Directory containing PDF files      (default: co2m/pdfs)
  --metadata-dir  Directory containing metadata files  (default: co2m/metadata)
  --out           Output directory for artifacts       (default: co2m/artifacts)
  --config        Path to concepts.yaml               (default: config/concepts.yaml)
```

## Technology stack

| Library | Role |
|---------|------|
| [PyMuPDF](https://pymupdf.readthedocs.io/) | PDF text extraction |
| [pandas](https://pandas.pydata.org/) | Tabular data processing |
| [pyarrow](https://arrow.apache.org/docs/python/) | Parquet I/O |
| [scikit-learn](https://scikit-learn.org/) | TF-IDF keyword extraction |
| [PyYAML](https://pyyaml.org/) | Taxonomy config parsing |
| [Streamlit](https://streamlit.io/) | Dashboard UI |
| [Plotly](https://plotly.com/python/) | Interactive charts |
